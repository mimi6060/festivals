# Méthodes d'Élicitation Avancée - Référence

Ce document liste toutes les méthodes disponibles pour approfondir la réflexion sur un projet.

---

## Collaboration

| # | Méthode | Description | Pattern |
|---|---------|-------------|---------|
| 1 | **Stakeholder Round Table** | Réunir plusieurs personas pour apporter des perspectives diverses - essentiel pour la collecte de requirements et trouver des solutions équilibrées | perspectives → synthèse → alignement |
| 2 | **Expert Panel Review** | Assembler des experts du domaine pour une analyse approfondie - idéal quand la profondeur technique est nécessaire | avis experts → consensus → recommandations |
| 3 | **Debate Club Showdown** | Deux personas argumentent des positions opposées avec un modérateur - excellent pour explorer les décisions controversées | thèse → antithèse → synthèse |
| 4 | **User Persona Focus Group** | Réunir les personas utilisateurs pour réagir aux propositions et partager leurs frustrations - essentiel pour valider les features | réactions → préoccupations → priorités |
| 5 | **Time Traveler Council** | Le toi du passé et du futur conseillent le toi du présent - puissant pour la perspective long terme vs court terme | sagesse passée → choix présent → impact futur |
| 6 | **Cross-Functional War Room** | PM + ingénieur + designer s'attaquent à un problème ensemble - révèle les compromis faisabilité/désirabilité/viabilité | contraintes → compromis → solution équilibrée |
| 7 | **Mentor and Apprentice** | Expert senior enseigne à un junior qui pose des questions naïves - fait émerger les hypothèses cachées | explication → questions → compréhension profonde |
| 8 | **Good Cop Bad Cop** | Persona encourageant et persona critique alternent - trouve les forces et faiblesses | encouragement → critique → vue équilibrée |
| 9 | **Improv Yes-And** | Plusieurs personas construisent sur les idées des autres sans bloquer - génère des directions créatives inattendues | idée → construction → construction → surprise |
| 10 | **Customer Support Theater** | Client mécontent et support jouent un scénario - révèle les vraies frustrations utilisateurs | plainte → investigation → résolution → prévention |

## Avancé

| # | Méthode | Description | Pattern |
|---|---------|-------------|---------|
| 11 | **Tree of Thoughts** | Explorer plusieurs chemins de raisonnement simultanément puis évaluer - parfait pour problèmes complexes avec plusieurs approches valides | chemins → évaluation → sélection |
| 12 | **Graph of Thoughts** | Modéliser le raisonnement comme un réseau interconnecté d'idées - idéal pour la pensée systémique | nœuds → connexions → patterns |
| 13 | **Thread of Thought** | Maintenir un raisonnement cohérent sur de longs contextes - essentiel pour la consistance | contexte → fil → synthèse |
| 14 | **Self-Consistency Validation** | Générer plusieurs approches indépendantes puis comparer - crucial pour décisions à enjeux élevés | approches → comparaison → consensus |
| 15 | **Meta-Prompting Analysis** | Prendre du recul pour analyser la structure de l'approche elle-même | actuel → analyse → optimisation |
| 16 | **Reasoning via Planning** | Construire un arbre de raisonnement guidé par des modèles et états-buts - excellent pour planification stratégique | modèle → planification → stratégie |

---

## Compétitif

| # | Méthode | Description | Pattern |
|---|---------|-------------|---------|
| 17 | **Red Team vs Blue Team** | Analyse adversariale attaque-défense pour trouver les vulnérabilités - critique pour tests de sécurité | défense → attaque → renforcement |
| 18 | **Shark Tank Pitch** | Entrepreneur pitche à des investisseurs sceptiques - stress-test la viabilité business | pitch → challenges → raffinement |
| 19 | **Code Review Gauntlet** | Devs seniors avec différentes philosophies reviewent le même code - fait émerger les débats de style | reviews → débats → standards |

## Technique

| # | Méthode | Description | Pattern |
|---|---------|-------------|---------|
| 20 | **Architecture Decision Records** | Plusieurs personas architectes proposent et débattent des choix avec compromis explicites | options → compromis → décision → rationnel |
| 21 | **Rubber Duck Debugging Evolved** | Expliquer son code à des canards progressivement plus techniques jusqu'à trouver le bug | simple → détaillé → technique → eureka |
| 22 | **Algorithm Olympics** | Plusieurs approches compétitionnent sur le même problème avec benchmarks | implémentations → benchmarks → gagnant |
| 23 | **Security Audit Personas** | Hacker + défenseur + auditeur examinent le système depuis différents modèles de menace | vulnérabilités → défenses → conformité |
| 24 | **Performance Profiler Panel** | Expert BDD + spécialiste frontend + DevOps diagnostiquent les lenteurs | symptômes → analyse → optimisations |

## Créatif

| # | Méthode | Description | Pattern |
|---|---------|-------------|---------|
| 25 | **SCAMPER Method** | Appliquer 7 lentilles créatives (Substituer/Combiner/Adapter/Modifier/Proposer/Éliminer/Renverser) | S→C→A→M→P→E→R |
| 26 | **Reverse Engineering** | Travailler à rebours depuis le résultat souhaité pour trouver le chemin | état final → étapes arrière → chemin avant |
| 27 | **What If Scenarios** | Explorer des réalités alternatives pour comprendre les possibilités | scénarios → implications → insights |
| 28 | **Random Input Stimulus** | Injecter des concepts non-reliés pour créer des connexions inattendues | mot aléatoire → associations → idées nouvelles |
| 29 | **Exquisite Corpse Brainstorm** | Chaque persona ajoute à l'idée en ne voyant que la contribution précédente | contribution → passage → contribution → surprise |
| 30 | **Genre Mashup** | Combiner deux domaines non-reliés pour trouver des approches fraîches | domaine A + domaine B → insights hybrides |

## Recherche

| # | Méthode | Description | Pattern |
|---|---------|-------------|---------|

| 32 | **Thesis Defense Simulation** | Étudiant défend son hypothèse contre un comité - stress-test la méthodologie | thèse → challenges → défense → raffinements |
| 33 | **Comparative Analysis Matrix** | Plusieurs analystes évaluent les options contre des critères pondérés | options → critères → scores → recommandation |

## Risque

| # | Méthode | Description | Pattern |
|---|---------|-------------|---------|
| 34 | **Pre-mortem Analysis** | Imaginer l'échec futur puis travailler à rebours pour le prévenir - puissant pour mitigation des risques | scénario d'échec → causes → prévention |
| 35 | **Failure Mode Analysis** | Explorer systématiquement comment chaque composant pourrait échouer | composants → échecs → prévention |
| 36 | **Challenge from Critical Perspective** | Jouer l'avocat du diable pour stress-tester les idées | hypothèses → challenges → renforcement |
| 37 | **Identify Potential Risks** | Brainstormer ce qui pourrait mal tourner dans toutes les catégories | catégories → risques → mitigations |
| 38 | **Chaos Monkey Scenarios** | Casser délibérément les choses pour tester la résilience | casser → observer → renforcer |

## Core

| # | Méthode | Description | Pattern |
|---|---------|-------------|---------|
| 39 | **First Principles Analysis** | Éliminer les hypothèses pour reconstruire depuis les vérités fondamentales - technique de percée pour l'innovation | hypothèses → vérités → nouvelle approche |
| 40 | **5 Whys Deep Dive** | Demander "pourquoi" répétitivement pour creuser jusqu'aux causes racines | chaîne de pourquoi → cause racine → solution |
| 41 | **Socratic Questioning** | Utiliser des questions ciblées pour révéler les hypothèses cachées | questions → révélations → compréhension |
| 42 | **Critique and Refine** | Review systématique pour identifier forces et faiblesses puis améliorer | forces/faiblesses → améliorations → raffiné |
| 43 | **Explain Reasoning** | Parcourir la réflexion étape par étape pour montrer comment les conclusions ont été atteintes | étapes → logique → conclusion |
| 44 | **Expand or Contract for Audience** | Ajuster dynamiquement le niveau de détail pour l'audience cible | audience → ajustements → contenu raffiné |
